# KlevinPlan

Учебный прототип веб-приложения для совместного планирования дел и событий.

## Состав проекта

- `server.js` — серверная часть на Node.js и Express;
- `database.js` — подключение SQLite, создание таблиц и операции чтения/сохранения данных;
- `database.sqlite` — файл базы данных SQLite;
- `public/index.html` — основная HTML-страница и стили интерфейса;
- `public/app.js` — клиентская логика приложения;
- `package.json` — зависимости и команды запуска.

## Запуск локально

```bash
npm install
npm start
```

После запуска приложение доступно по адресу:

```text
http://localhost:3000
```

## Хранение данных

В приложении используется SQLite. Основной файл базы данных — `database.sqlite`, работа с ним вынесена в `database.js`.

ER-диаграмма в отчёте показывает проектную реляционную модель: `users`, `spaces`, `space_members`, `tasks`, `events`, `notes`, `comments`, `attachments`.

В коде также используются служебные таблицы для приглашений, запросов в друзья и технических полей интерфейса. Они нужны для работы прототипа и не изменяют основную ER-модель.
